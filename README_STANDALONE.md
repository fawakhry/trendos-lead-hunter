# صياد العملاء - نسخة مستقلة فعّالة

هذه النسخة تعمل بدون TrendOS وبدون Backend.

## التشغيل السريع
1. ارفع الملفات الأربعة في جذر GitHub Repo: `index.html`, `styles.css`, `app.js`, `config.js`.
2. فعل GitHub Pages من Settings > Pages.
3. افتح الرابط، واضغط بيانات تجربة.
4. أضف مصادر فيسبوك والكلمات الساخنة.
5. عند العثور على عميل، انسخ النص والرابط واحفظ الفرصة.

## التخزين
البيانات تحفظ داخل متصفحك باستخدام LocalStorage. استخدم زر نسخة احتياطية JSON لتصدير نسخة دورية.

## إضافة Chrome
افتح Chrome Extensions > Developer mode > Load unpacked واختر مجلد `chrome-extension`.
ضع رابط صفحة GitHub Pages في خانة رابط برنامج صياد العملاء.
بعدها حدد نص كومنت/بوست من فيسبوك واضغط فتح في صياد العملاء.


## تحديث V2 - المراقبة الآمنة كل 10/15 دقيقة

هذه النسخة تضيف لوحة "المراقبة الآمنة".

ما الذي يعمل فعليًا:
- تضيف لينكات الجروبات والصفحات كمصادر.
- تحدد مدة مراجعة لكل مصدر: 10 دقائق أو 15 دقيقة أو أكثر.
- البرنامج يراجع المواعيد محليًا داخل المتصفح.
- عند حلول موعد مصدر، يظهر في "مصادر مستحقة للمراجعة".
- زر "افتح أول بحث مستحق" يفتح بحث فيسبوك داخل الجروب نفسه إذا كان الرابط من نوع facebook.com/groups/ID.
- يمكن تفعيل "فتح البحث تلقائيًا" لفتح أول بحث مستحق في تبويب جديد، حسب قيود المتصفح.
- إضافة Chrome تحفظ النص المحدد واللينك داخل البرنامج، ثم البرنامج يحلل النص ويقترح الرد.

ما الذي لا يفعله النظام:
- لا يدخل فيسبوك ويسحب البوستات أو التعليقات تلقائيًا.
- لا يقرأ محتوى الجروبات من غير تدخل منك.
- لا يرد تلقائيًا.

السبب: سحب بيانات فيسبوك أوتوماتيك من الجروبات بدون API رسمي قد يعرّض الحساب أو الصفحة للحظر.
